# 项目概述

## 什么是 CampusWorld

CampusWorld 是新一代智慧园区操作系统，基于原生 AI 架构设计。它借鉴 MUD（Multi-User Dungeon）世界的设计原理，以图语义结构构建知识本体，实现物理世界、数字系统、设备和人的语义孪生与交互映射。

当前仓库采用**智能模块化单体**实现：后端在同一进程与代码库内组织 API、SSH、命令系统、图本体、世界包和 Agent Runtime 等模块，通过清晰边界保持可演进性。微服务拆分、服务发现、服务网格等属于未来演进选项，不是当前默认运行形态。

## 核心理念

### 统一语义世界

传统园区系统以功能模块为核心，而 CampusWorld 以"世界"为核心，把空间、系统、设备、人员、任务和 Agent 放入同一个语义世界：
- 万物皆对象，万物皆节点，世界由节点与关系共同构成
- 建筑、楼层、房间、设备、用户、任务和 Agent 都是可查询、可推理、可交互的世界对象
- Agent 是语义世界中的自主行动者，既消费知识，也通过能力改变世界状态

### 知识表达

知识服务负责维护语义世界中的知识对象与关系：
- 事实类知识表达空间、设备、环境、人员等现实状态
- 概念类知识表达目标、规则、流程、经验、行动和 Skill
- 图模型统一表达对象、关系、空间层级和世界拓扑
- 知识服务为 Agent 构建记忆、语义上下文、查询和推理结果

### 能力表达

能力即工具。CampusWorld 通过 command/tool 表达可执行能力：
- Agent 是主要交互主体，命令是语义世界提供给 Agent 的工具能力
- 命令系统是协议中立的能力层，多种接入方式共享同一套能力语义
- 人类用户可以通过终端、API 或前端入口触发这些能力，也可以授权 Agent 自主使用能力
- 能力作用于语义世界对象，用于查询、移动、进入世界、触发任务或控制设备
- LinkSoft 是后续需集成的能力组件，用于连接设备、真实系统和外部 IT 系统
- 能力执行过程和结果可反哺知识服务，形成持续更新的语义数字孪生

### Agent 与系统分层

CampusWorld 以三层职责组织业务类 OS 能力：系统适配层负责公共能力和外部接入，知识与能力层负责语义世界和工具能力，Agent 服务层负责 Agent 构建、运行和协同。

### 分层架构

```
Agent 服务层（AI 使能/Agent 交互）
    ↓
知识与能力层（知识服务/能力服务）
    ↓
系统适配层（公共服务/设备接入）
```

这三层是概念职责划分，不代表当前按独立微服务拆分部署。当前实现以内聚的后端单体承载领域模块，多种接入方式共享命令与服务逻辑。

## 技术架构

### 后端

- **FastAPI**：高性能异步 Web 框架，提供 REST API 和 OpenAPI 文档
- **SQLAlchemy**：Python ORM，支持复杂查询和数据库迁移
- **Paramiko**：SSH 服务器实现，支持终端连接
- **Graph Models**：基于图结构的领域模型（User、Character、Room、Building、World 等）
- **Commands**：语义世界提供给 Agent 的工具能力，同时支持 SSH、HTTP/WebSocket、CLI 和前端等入口复用

### 前端

- **Vue 3** + **TypeScript**：类型安全的前端开发
- **Vite**：快速的开发服务器和构建工具
- **Element Plus**：企业级 UI 组件库
- **Pinia**：轻量级状态管理

### 数据层

- **PostgreSQL**：主数据库，支持向量、时序、GIS 扩展
- **Redis**：缓存和会话存储
- **Docker Compose**：开发环境容器化

## 核心模块

| 模块 | 路径 | 说明 |
|------|------|------|
| SSH 服务器 | `backend/app/ssh/` | 基于 Paramiko 的 SSH 服务器，处理终端连接 |
| 命令系统 | `backend/app/commands/` | 继承自 `BaseCommand` 的命令体系，支持自动注册 |
| 引擎 | `backend/app/game_engine/` | 逻辑核心，支持多内容包 |
| 数据模型 | `backend/app/models/` | 图节点、语义关系与空间拓扑模型，支持动态发现 |
| 协议处理 | `backend/app/protocols/` | HTTP/WebSocket、SSH、CLI 等协议入口共享命令逻辑 |
| 核心模块 | `backend/app/core/` | 配置、日志、安全、数据库连接等基础功能 |

## 与传统园区系统的区别

| 特性 | 传统园区系统 | CampusWorld |
|------|-------------|-------------|
| 数据模型 | 关系型表格，按功能模块隔离 | 图结构，万物互联 |
| 交互主体 | 人通过表单、按钮、API 调用系统 | Agent 使用语义世界提供的工具能力，人通过多种入口参与或授权 |
| 扩展方式 | 新模块、新表、代码修改 | 新对象类型、新关系、新命令 |
| 设备语义 | 独立设备清单 | 设备作为世界对象，参与空间拓扑 |
