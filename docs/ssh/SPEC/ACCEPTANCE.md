# 奇点屋入口验收文档

## 目标

验证“奇点屋（SingularityRoom）作为系统默认登录入口 + 多世界连接枢纽”的规范是否成立，并覆盖成功、恢复、拒绝、降级四类路径。

## 统一前置条件

- SSH 服务可连接（默认 `localhost:2222`）。
- 存在可登录账号（例如 `admin/dev/campus`）。
- 数据库中存在 `SingularityRoom` 根节点（`type_code='room'` 且 `attributes.is_root=true`）。
- `campus_life` 世界可加载（用于世界内入口验收）。

## 场景 A：成功路径（新用户/无恢复状态）

### 步骤
1. 使用 SSH 登录。
2. 完成认证后观察用户当前位置。
3. 执行进入 `campus_life` 世界动作（命令或路由策略触发）。

### 期望
- 认证后先进入 `SingularityRoom`。
- 进入 `campus_life` 后，世界内默认位置为 `campus`。
- 会话保持 ACTIVE，可继续执行命令。

## 场景 B：恢复路径（Last Location Resume）

### 步骤
1. 用户在某世界内移动到非默认位置并离线。
2. 重新登录，按策略启用恢复。

### 期望
- 若恢复位置有效且有权限：恢复到最后位置。
- 若恢复位置无效：回退到 `SingularityRoom`。

## 场景 C：拒绝路径（无权限世界）

### 步骤
1. 用户登录后尝试进入无权限世界。

### 期望
- 返回明确拒绝信息（权限不足）。
- 用户保留在 `SingularityRoom`，会话不中断。

## 场景 D：降级路径（目标世界不可用）

### 步骤
1. 下线或禁用目标世界。
2. 用户登录后尝试进入该世界。

### 期望
- 返回“世界不可用”提示。
- 用户留在 `SingularityRoom`。
- 会话不中断，允许重新选择目标世界。

## 观察点与证据

- **日志**：认证成功/失败、spawn、世界路由结果、失败原因。
- **位置状态**：用户 `location_id/home_id` 与会话返回位置一致。
- **稳定性**：失败场景不应导致连接断开或服务器崩溃。

## 最小回归清单

- [ ] SSH 登录成功后进入 `SingularityRoom`。
- [ ] 从 `SingularityRoom` 进入 `campus_life` 后到 `campus`。
- [ ] 无权限世界请求被拒绝且会话保持。
- [ ] 世界不可用时回退到 `SingularityRoom`。
- [ ] 断线重连时恢复策略按预期生效。
# 验收检查表

## 连接验收

- [ ] `ssh localhost -p 2222` 建立 SSH 连接
- [ ] 连接建立后显示欢迎 Banner
- [ ] 服务器监听 0.0.0.0:2222（可配置）
- [ ] 支持 RSA 4096 位主机密钥

## 认证验收

- [ ] 正确用户名密码登录成功
- [ ] 错误密码返回 "Authentication failed"
- [ ] 连续 5 次失败后，锁定 IP 300 秒
- [ ] 锁定期间连接被拒绝

## 会话验收

- [ ] 登录成功后 spawn 到 SingularityRoom
- [ ] 会话超时（auth_timeout）后自动断开
- [ ] 正确执行 `quit` 断开连接
- [ ] 连接断开后释放所有资源

## 命令交互验收

- [ ] `look` 返回当前位置完整信息
- [ ] `who` 返回当前在线用户列表
- [ ] `help` 返回可用命令列表
- [ ] `time` 返回当前时间

## 安全验收

- [ ] 速率限制生效（10 连接/分钟）
- [ ] 登录失败追踪记录准确
- [ ] IP 白名单功能正常
- [ ] 会话数据不泄露到其他会话

## 性能验收

- [ ] 单会话内存占用 < 10MB
- [ ] 50 并发会话正常运行
- [ ] 命令执行响应 < 100ms（不含网络延迟）
